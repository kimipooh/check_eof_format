# EOF Format Checker

## 概要

`check_eof_format.py` は、テキストファイルやソースコードファイルのファイル末尾の状態を検査するコマンドラインツールです。

**このプログラムは検出のみを行います。対象ファイルの内容は一切変更しません。**

次の 2 種類の問題を検出します。

- **末尾改行なし** — ファイルの最後が改行文字 `\n` で終わっていない
- **末尾空白行あり** — ファイル末尾に空白文字だけで構成された余分な行が存在する

末尾改行だけでなく、末尾の空白行も検出するため、ファイル末尾全般の書式チェッカーとして機能します。

## 用途

Python などのソースコードや一般的なテキストファイルにおいて、ファイル末尾の書式問題を一括チェックするために利用できます。

- 授業・課題提出物の一括検査（末尾改行なし・末尾の余分な空白行の検出）
- コードレビュー前の事前確認
- CI/CD パイプラインでのファイル品質チェック

## 動作環境

- Python 3.x
- 標準ライブラリのみ使用（追加インストール不要）

## 使い方

### 対話モード

引数なしで実行すると、チェック対象のファイル名または glob パターンの入力を求められます。

```bash
python3 check_eof_format.py
```

入力例：

```text
sample/*.py
```

ファイル名に空白が含まれる場合は引用符で囲みます。

```text
'sample/ex 01.py'
```

### 引数モード

コマンド引数としてファイル名または glob パターンを直接指定することもできます。

```bash
python3 check_eof_format.py 'sample/*.py'
```

複数のパターンを並べて指定することも可能です。

```bash
python3 check_eof_format.py 'src/*.py' 'tests/*.py'
```

## 検出の設計

このプログラムは、対象ファイルをバイナリデータとして読み込み、ファイル末尾の状態を分類します。文字コードを解釈しないため、日本語ファイル名や日本語コメントを含むファイルでも、末尾判定自体は文字コードに依存しません。

処理の流れは次のとおりです。

1. 入力されたファイル名または glob パターンを受け取る
2. `glob.glob()` で実際のファイル一覧に展開する
3. 各ファイルをバイナリモードで開く
4. ファイル全体をバイト列として読み込む
5. 最後の 1 バイトが `\n` かどうかを確認する（末尾改行の有無）
6. ファイル末尾に空白だけの行が続いていないか確認する（末尾空白行の検出）
7. 問題の種類ごとにファイル名を出力する

## 判定ルール

### 末尾改行なし

最後の 1 バイトが改行文字 `\n` の場合、そのファイルは末尾改行ありと判定します。

最後の 1 バイトが `\n` 以外の場合、そのファイルは末尾改行なしとして検出します。

空ファイルは、最後の改行文字が存在しないため、末尾改行なしとして扱います。

### 末尾に空白だけの余計な行がある

ファイルを行単位に分け、末尾側から順に確認します。末尾にある行が空行またはスペース・タブなどの空白文字だけで構成されている場合、「末尾に空白だけの余計な行がある」と判定します。

検出対象となる末尾の例：

```text
print(list2)
    
```

上の例では、`print(list2)` の次の行に空白文字だけが入っています。

末尾に複数の空白行がある場合は、その行数も表示します。

## 出力内容

問題が見つかった場合は、種類ごとにファイルパスを表示します。

- `[Missing final newline]` — ファイルの最後が改行で終わっていないファイルの一覧
- `[Trailing whitespace-only line]` — 末尾に空白だけの余計な行があるファイルの一覧（行数付き）

同じファイルが両方の問題に該当する場合は、両方の欄に表示されます。該当するファイルがない種類の見出しは表示されません。

最後に、確認したファイル数と問題の種類ごとの件数を表示します。

**出力例：**

```text
[Missing final newline]
sample/ex11111.py

[Trailing whitespace-only line]
sample/ex11112.py (1 trailing line(s))

Checked: 48 file(s)
Missing final newline: 1 file(s)
Trailing whitespace-only line: 1 file(s)
```

この例では、`sample/ex11111.py` に末尾改行がなく、`sample/ex11112.py` の末尾には空白だけの行が 1 行あることを示しています。

`(1 trailing line(s))` は末尾にある空白だけの行数を表します。たとえば `(2 trailing line(s))` であれば、空白だけの行が 2 行続いていることを意味します。

`Checked` は確認対象になったファイル数です。`Missing final newline` と `Trailing whitespace-only line` の件数は、それぞれの問題に該当したファイル数です。

## 終了コード

| コード | 意味 |
|--------|------|
| `0` | 問題は見つからなかった |
| `1` | 末尾改行なし、または末尾の空白だけの行が見つかった |
| `2` | 入力がない、または対象ファイルが見つからなかった |

終了コードを利用することで、CI/CD パイプラインやシェルスクリプトとの組み合わせた自動チェックが可能です。

## 注意点

- このプログラムは**検出のみ**を行います。対象ファイルの内容を修正したり、末尾改行を自動追加したり、余分な空白行を削除したりはしません。
- glob パターンはシェルではなく Python の `glob.glob()` で展開されます。シェルがパターンを先に展開する場合があるため、引数として渡す際はシングルクォートで囲むことを推奨します。
- Windows 環境での `\r\n` 改行については、末尾が `\n` で終わっていれば末尾改行ありと判定します（`\r\n` も `\n` で終わるため）。
